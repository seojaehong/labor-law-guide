# 취업규칙 검토 스킬 × 법률MCP 통합 설계

## 현황

### 이미 구축된 것
- **korean-law MCP 서버** (v2.2.0): `https://korean-law-mcp.fly.dev/mcp`
  - settings.json에 등록 완료
  - 법제처 API 기반 법령 조회
  - 환경변수: `LAW_OC=iceamericano9`

- **취업규칙 검토 스킬**: `~/.claude/skills/employment-rules-review.md`
  - 7단계 워크플로우
  - 표준취업규칙(2026) 기준 비교 검토
  - 레퍼런스 파일 5종 세팅 완료

### 연동 아키텍처

```
사용자 → "취업규칙 검토해줘" + 파일 첨부
         ↓
    [employment-rules-review 스킬 트리거]
         ↓
    Step 1: 파일 로드 + 표준취업규칙 로드
         ↓
    Step 2-4: 조문 매핑 + 필수기재사항 + 상세 비교
         ↓  (표준취업규칙 기준으로 1차 검토)
         ↓
    Step 4 보충: korean-law MCP 교차검증
         ├── 법령 조문 최신 확인 (조문이 개정되었을 수 있음)
         ├── 행정해석 검색 (해석이 달라진 사례)
         └── 관련 판례 확인 (실무상 쟁점)
         ↓
    Step 5-7: 검토 결과 + 보고서 출력
```

## 법률MCP 활용 시나리오

### 시나리오 1: 법령 조문 최신 확인
표준취업규칙은 2026년 2월 기준이므로, 그 이후 개정된 법령이 있을 수 있다.
```
검토 중 근로기준법 제60조 인용 시
→ korean-law MCP로 현재 시행 중인 제60조 조문 확인
→ 표준취업규칙 기재 내용과 상이한 경우 알림
```

### 시나리오 2: 판례 기반 해석 보충
```
통상임금 관련 조항 검토 시
→ 대법원 2024.12.19. 전원합의체 판결 내용 확인
→ 해당 사업장의 수당 체계가 판례 변경에 영향받는지 분석
```

### 시나리오 3: 행정해석 참조
```
육아휴직 관련 조항에서 "같은 자녀" 해석이 모호한 경우
→ 고용노동부 행정해석 검색
→ 구체적 사례에 대한 해석 근거 제시
```

## 주의사항

1. **기본은 표준취업규칙**: MCP는 보충적 교차검증에만 사용. 1차 검토 근거는 항상 표준취업규칙
2. **MCP 실패 시 graceful**: 법률MCP 접속 불가해도 기본 검토는 정상 수행
3. **인용 투명성**: MCP로 확인한 내용은 출처를 명시 (예: "법제처 API 2026.4.1. 기준")
4. **비용 관리**: MCP 호출은 필요시에만 (모든 조문에 대해 일괄 호출하지 않음)
